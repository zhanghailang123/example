### 流程引擎
可视化的配置
清晰的业务语义表达
业务逻辑的解耦

实现效果
> 应用启动读取各个工作流的配置
> 这时请求进来根据业务类型分发工作流的配置 dispatch
> 执行工作流 ==》需要考虑执行的时机 发起执行方是谁
> 工作流的流转 审批流的流转  
> 
那么我们怎么来设计呢
    1.流程化的配置通过XML或者Json文件的形式进行配置
    2.抽象流程为processDefinaion
    3.抽象流程节点为 processNode
    4.抽象上下文为 processContext 
        4.1 processContext来控制方法的执行吗
